# Blade visualization outputs

Generated by [`../tools/blade_viz.py`](../tools/blade_viz.py) directly from the
AeroDyn15 model files, so they stay in sync with the data. Re-run after editing
the model:

```bash
python archive/tools/blade_viz.py              # writes the files below
python archive/tools/blade_viz.py --snapshots  # also (re)writes the PNG snapshots
python archive/tools/blade_viz.py --snapshots --scale 6  # higher-resolution snapshots
```

The PNG snapshots of the Plotly views are rendered on a fixed logical canvas
(2200x1200 for the interactive view) and their resolution is set by `--scale`
(default 4, giving `blade_interactive.png` at 8800x4800). Use `--scale` to go
higher: it multiplies pixel density and keeps the text proportional, whereas
inflating the canvas width/height would just make the fonts look tiny.

| File | What it shows |
|------|---------------|
| `blade_overview.png` | Static multi-panel figure, all aligned on the span axis: planform coloured by airfoil ID (`BlAFID`), the airfoil-ID-vs-span step plot, the **distance between blade nodes (Δspan)** with a node-position rug, derived relative thickness `t/c`, and twist. Every point is labelled with its `BlAFID`; each Δspan gap is labelled with the adjacent airfoil pair (e.g. `5→6`), since a gap belongs to two airfoils rather than one. |
| `blade_interactive.html` | Plotly 2D explorer with the same panels and labels (Δspan as a bar chart). Hover any node to read its airfoil ID, name, `Re`, chord, twist, `t/c` and the gap to the next node. Best for finding *which* airfoil sits at a given span. |
| `blade_3d.html` | Rotatable 3D loft. Each spanwise section is drawn in a **single flat colour** (its airfoil ID, with no interpolation between sections), so the band heights also read as the node spacing. |
| `blade_interactive.png`, `blade_3d.png` | Static snapshots of the two HTML views (so they preview on GitHub). |
| `uniform_validation.png` | OpenFAST 3.5.5 validation of the uniform blended blade (`AeroDyn15_uniform/`) against the original: rotor power/thrust/torque and spanwise loads agree to well under 1% in the loaded span. |

### Caveats
- The polar files contain **no shape coordinates** (`NumCoords 0`), so the 3D
  loft uses a **generic NACA-type section scaled to a derived `t/c`**: it is
  illustrative of planform/taper/twist, not exact section geometry.
- `t/c` is **inferred from the airfoil naming convention** (Cylinder = 100 %,
  `EC1-45` = 45 % down to `EC1-16` = 16 %, linearly blended by the leading
  fraction in each name). Adjust `BASE_TC` in `blade_viz.py` if your convention
  differs.
